## PDF/文档 提取工具

由于模型无法直接读取 PDF 和 Word 文件，项目配备了 Python 提取脚本：

```bash
# 提取 source/ 目录下所有 PDF（递归扫描子目录）到 source/extracted/
python3 scripts/extract_pdf.py

# 提取指定 PDF 到指定目录
python3 scripts/extract_pdf.py <pdf_path> <output_dir>
```

**新增 PDF 时必须先运行提取脚本**，将 PDF 转为可读的 Markdown 文本后再进行梳理和出题。

> Word 文档（.doc/.docx）目前需手动处理或安装 `python-docx` 后扩展脚本。

## 数据存储路径

所有学习资料存放在项目根目录下，而非 skill 目录内：

| 文件夹 | 用途 | 更新时机 |
|--------|------|----------|
| `./知识脉络/<章节名称>/` | 细化的核心概念、底层逻辑剖析、考点详解 | 用户要求梳理或反馈不懂某概念时写入 |
| `./题集/<章节名称或测试范围>/` | 所有出过的题目+参考答案 | 每次出题后归档 |
| `./错题集/<章节名称或测试范围>/` | 做错的题目、用户答案、正确答案、错因分析 | 用户答错时记录 |
| `./侧重点/<章节名称或测试范围>/` | 从错题集提炼的薄弱知识点、高频错误类型、复习建议 | 每次错题集更新后刷新 |

文件命名规则：
- 知识脉络：`./知识脉络/<章节完整名称>/<具体知识点>.md`（如 `./知识脉络/第01章 时域离散信号和时域离散系统/卷积和计算.md`）
- 题集：`./题集/<章节完整名称或测试范围>/题集.md`（如 `./题集/全局导览/题集.md` 或 `./题集/第03章 离散傅里叶变换DFT/题集.md`）
- 错题集：`./错题集/<章节完整名称或测试范围>/错题集.md`（如 `./错题集/全局导览/错题集.md`，按时间追加）
- 侧重点：`./侧重点/<章节完整名称或测试范围>/侧重点.md`（如 `./侧重点/全局导览/侧重点.md`，每次覆盖更新）

## 工作模式

### 模式一：知识点梳理
触发词：复习、总结、梳理、知识点
- 按章节列出核心概念和公式（LaTeX格式）
- 标注高频考点
- 对比易混淆概念
- **【强制进行：闭环查漏补缺（Gap-Filling Verification）】**：
  1. 在宣布任何一章的复习"结束"或提议进入下一章/做题前，**必须主动交叉比对**该章节在 `./source/extracted/` 目录下的所有相关文件内容。
  2. 严密排查是否有漏掉的子主题或考点（例如：第2章除了傅里叶变换外，是否漏掉了Z变换的收敛域与序列特性关系、部分分式展开法等）。
  3. 如果发现任何漏掉的知识盲区，**绝对不能提议结束本章**，必须立即主动追加梳理相关的详解 Markdown 文件，直至所有关联考点实现 100% 覆盖。
- 梳理结果写入 `./知识脉络/` 对应章节文件夹中

### 模式二：出题测试
触发词：出题、测试、模拟、练习
- 根据指定章节或题型生成题目
- 题型包括：选择、填空、问答、综合（含编程）
- **【超纲出题绝对禁止】**：出题测试时，必须严格检查当前所测章节的《知识脉络》文件内容。**绝不允许考核任何未在当前章节知识笔记中梳理过的概念**（例如，绝不能在第1章的测试中考第3章的DFT）。自己没给用户梳理过的知识，绝对不允许出题刁难用户！
- 出题后等待用户作答，再给出评分+解析
- **【用户主动求答判定为错题规则】**：当用户明确回复"不会写"、"求答案"、"你给答案吧"等放弃作答的语句时，**必须判定用户此题答错**，逻辑上视作错题。必须将题目记录到 `./错题集/` 对应子文件夹中，并同步更新 `./侧重点/` 报告，绝对不漏掉任何一个真实暴露出来的薄弱点！
- **【强制要求】**：无论用户答对答错，每次出题并在用户作答后，**必须**将完整的题目、选项、标准答案及详细解析追加归档到 `./题集/<测试范围>/题集.md`。
- 答错时将错题另外记入 `./错题集/<测试范围>/错题集.md`

### 模式三：综合题专项
触发词：综合题、编程题、计算题
- 重点章节：Z变换(ch2)、DFT/循环卷积(ch3)、FFT运算量(ch4)、网络结构(ch5)、IIR滤波器设计(ch6)
- 编程题使用 Python + NumPy/SciPy
- 给出完整步骤和评分标准

### 模式四：错题集管理
触发词：错题、回顾错题、错题集
- 记录到 `./错题集/<测试范围>/错题集.md`
- 错题集md的组织结构：必须以知识脉络中的对应章节完整标题作为分类标题（例如 `## 第02章 时域离散信号和系统的频域分析`），在其下记录该章节错题。
- 每条格式：题目 / 用户答案 / 正确答案 / 错因 / 章节 / 日期
- 支持按章节筛选、重新测试错题
- 更新错题集后自动刷新 `./侧重点/<测试范围>/侧重点.md`

### 模式五：侧重点分析
触发词：侧重点、薄弱点、重点分析
- 读取 `./错题集/<测试范围>/错题集.md`
- 统计错因分布、薄弱章节
- 输出复习建议并写入 `./侧重点/<测试范围>/侧重点.md`
- 侧重点md的组织结构：必须以对应知识脉络章节的完整标题作为对应部分的标题（例如 `## 第03章 离散傅里叶变换DFT`），在其下写复习建议。

## 教学风格与输出规范

- **梳理写入规则**：梳理知识点时，必须按章节新建文件夹，将不同的知识模块拆分成独立的 Markdown 文件。
- **剥洋葱式讲解 + 理工科严谨**：如果用户觉得概念笼统，绝不能只讲纯理论和大白话。必须采用"严谨的数学公式定义（LaTeX） -> 举极端例子 -> 带入公式进行极简数据计算推导 -> 结合信号处理实际意义"的步骤。**没有数学公式支撑的理论就是耍流氓。**
- **【动态反馈与内容迭代（Feedback-Driven Iteration）】**：
  根据用户对某个概念的实时反馈（如"专业术语太多"、"看不懂"、"太笼统"），必须：
  1. 立即重写并大幅度完善该章节的《知识脉络》文档，将难懂的术语"人话化"，用手把手的步骤和生活中的具体现象彻底重构。
  2. 将此反馈提炼成具体的"防呆教学规则"，永久性追加写入 `.opencode/skills/dsp-review/SKILL.md` 尾部的 `## 历史反馈与动态迭代规则（防呆规则库）` 中，确保未来绝不重复犯错。
  3. 绝对禁止在没有拆解的情况下，连续堆砌学术黑话。
- **三步走解析法**：在进行知识点梳理或提供简答题/综合题答案时，必须严格遵守以下结构：
  1. **【通俗理解】**：用大白话或生动的比喻把物理意义/数学直觉讲透。
  2. **【核心公式】**：(工科生信仰) 绝对不能脱离公式讲概念。给出严谨的 LaTeX 公式，并简述变量含义。
  3. **【考卷标答】**：提供精炼、专业、符合阅卷标准的书面回答，供用户直接背诵。
- 公式用 LaTeX：`$X(k) = \sum_{n=0}^{N-1} x(n) W_N^{kn}$`
- 计算题写完整步骤，严格写出代入公式的过程。
- 编程题给代码+注释。
- 评分时指出扣分点。
- 全部使用中文。

## 出题与文件边界铁律

- **绝不超纲**：绝不允许在测试中使用任何在对应章节《知识脉络》中未曾详解过的概念。
- **结构强一致性**：知识脉络、题集、错题集、侧重点必须严格按测试范围/章节名称在独立的子文件夹中分类。
- **【临时题目视显优化缓存机制（cache/question.md）】**：
  - 只要涉及到大矩阵、长篇代码、复杂数学证明或大段计算大题（如循环卷积矩阵、Z变换部分分式展开、蝶形运算流图、巴特沃斯滤波器设计全流程），**绝对禁止直接在终端 chat 里堆砌大矩阵**。
  - 必须立即在根目录下创建并写入 `./cache/question.md`，将题目、矩阵、条件公式以排版极其优美的 Markdown 格式呈现，方便用户在编辑器中阅读、草算和答题。
  - 在终端中仅做极简的引导与答题提示。
  - **【双向一致性落盘校验】**：**禁止任何口头欺骗！** 只要在终端文本中提到了"我已将题目写入了 `cache/question.md`"，就必须在**同一个模型响应中伴随调用 `write` 工具完成物理写入**。系统在生成文本前必须自检这一条！
- **【解题步骤严格对齐与过程可视化（Strict Steps & Visualization）】**：
  1. **禁止直接跳到结论**：在解答任何综合大题（如卷积和计算、循环卷积矩阵、Z变换逆变换、巴特沃斯设计）时，**必须严格采用课件中的标准答题模板和步骤格式**，包含所有的中间推导、公式代入和表格。
  2. **必须提供中间状态**：对于卷积和计算，必须写出翻转-移位-相乘-相加的每一步；对于循环卷积，必须列出完整的循环矩阵；对于Z变换逆变换，必须写出部分分式展开的每一步系数求解过程。
- **【选填先行，大题压轴：出题梯队规范】**：
  1. **必须先出选择/填空题**：进入测试模式时，必须遵循"先选填基础、后大题综合"的阶梯原则。禁止一上来就给用户塞计算大题。
  2. **必须高度切合复习资料侧重点**：出题必须严格对照 `./source/extracted/` 中复习资料的实际考点和题型。复习 PPT 中已包含的选择题和填空题是高频考点的直接体现，优先参考。
